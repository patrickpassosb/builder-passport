# UI Spec

## General design
- clean
- modern
- minimal
- web3 aesthetic without clutter
- purple accents inspired by Monad
- demo-first

## Page 1 — Landing `/`
### Sections
1. Hero
2. Problem
3. How it works
4. Passport preview
5. Why Monad
6. CTA

### Hero copy
Title:
Builder Passport

Subtitle:
Portable proof of participation, contribution and wins in hackathons.

Buttons:
- Connect Wallet
- View Demo Passport

### Problem section
3 cards:
- Wins get lost in social posts
- Contributions are invisible
- Builders without pedigree stay underrated

### How it works
3 steps:
- Join a hackathon
- Get attested by peers
- Build a portable passport

### Preview section
Show a mocked passport card with:
- display name
- 1 event joined
- 2 peer attestations
- 1 official award
- AI summary

## Page 2 — Hackathon page `/hackathon/[id]`
### Main blocks
- Event header
- Join hackathon button
- Participant list
- Attestation actions
- Organizer panel

### Attestation buttons
- Technical
- Product
- Pitch
- Helpful
- Teamwork

### Organizer actions
- Finalist
- Winner
- Honorable Mention
- Best Technical Solution

## Page 3 — Passport page `/passport/[address]`
### Main blocks
- Profile header
- Social links
- Events joined
- Attestations by category
- Official awards
- AI summary card

### Profile header should show
- displayName
- handle
- short bio
- wallet address
- GitHub link
- LinkedIn link

## UX rule
Make the Passport page the best-looking page in the app.
