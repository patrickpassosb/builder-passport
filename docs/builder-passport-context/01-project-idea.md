# Project Idea

## Name
Builder Passport

## One-line description
A portable onchain record of hackathon participation, contribution, and wins.

## Problem
Hackathon achievements are fragmented.
- Wins usually become social posts and disappear
- Real contributions are invisible
- Builders without strong credentials stay underrated
- Organizers and communities cannot easily preserve reputation across events

## Solution
Builder Passport gives each participant a public profile where they can:
- create a builder identity
- join a hackathon
- receive peer attestations such as Technical, Helpful, Product, Pitch, Teamwork
- receive official awards from an organizer such as Winner, Finalist, Honorable Mention
- display everything in a single public passport page

## Why this matters
This project turns informal hackathon performance into a portable and verifiable credential.

## Positioning
Do **not** position this as a Devpost replacement.
Position it as a **reputation layer for hackathons**.

Good framing:
- Devpost stores projects
- Builder Passport stores builder credibility

## Why Monad
The product depends on frequent, lightweight social interactions onchain:
- profile creation
- join event
- attest peer
- mark award

The hackathon demo argument is:
> hackathon reputation should be cheap, fast, and public.

## Demo thesis
A hackathon should not end as a forgotten LinkedIn post.
It should become a reusable public credential.
