# Build Order

## Phase 1 — Contract
Build the contract first.
Priorities:
- Profile
- Hackathon
- Join
- Attest
- Award
- Events

## Phase 2 — Static frontend
Build the three pages with mock data first.
- Landing
- Hackathon page
- Passport page

## Phase 3 — Contract integration
Connect the frontend to:
- createProfile
- joinHackathon
- attestContribution
- assignAward
- profile and event reads

## Phase 4 — AI summary
Add a minimal offchain summary based on the final passport state.

## Phase 5 — Polish
Only if time remains:
- empty states
- loading states
- better card layout
- QR code to passport page

## Things to avoid during the hackathon
- scope expansion
- generic social feed
- multi-hackathon platform features
- ranking engine
- advanced indexing
